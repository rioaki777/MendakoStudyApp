# 管理画面仕様

## 概要
ひらがなレッスンの問題文を管理者が登録・編集・削除するための画面。子ども向けアプリとは別の管理者専用画面として実装する。

## アクセス方法

メインアプリ内の隠しエントリーポイントから遷移する（例: ホーム画面でバージョン表示を5回タップ）。または、将来的にはデバッグビルドのみ表示するボタンを設ける。

## 画面構成

### 管理画面トップ

| 要素 | 説明 |
|------|------|
| 問題文一覧 | 登録済みのひらがな問題文をリスト表示 |
| 「新規追加」ボタン | 問題文追加ダイアログを開く |
| 各行の「編集」ボタン | 問題文編集ダイアログを開く |
| 各行の「削除」ボタン | 確認後に削除 |
| 各行の受講統計 | 受講回数・正解回数を表示 |

### 問題文の一覧表示

| フィールド | 内容 |
|-----------|------|
| 問題文テキスト | 例: 「いぬ」「あおい そら」 |
| 受講回数 | `attempt_count` |
| 正解率 | `correct_count / attempt_count * 100`% |
| 登録日 | `created_at` をフォーマットして表示 |

### 問題文追加・編集ダイアログ

| 要素 | 説明 |
|------|------|
| テキスト入力フィールド | ひらがな文字列を入力（ひらがな・スペースのみ許可） |
| 「保存」ボタン | DBに INSERT / UPDATE |
| 「キャンセル」ボタン | 変更を破棄 |

## バリデーション

| ルール | 理由 |
|--------|------|
| ひらがな文字（と半角スペース）のみ許可 | 書き順データが存在しない文字の混入を防ぐ |
| 1文字以上20文字以下 | 画面に収まる長さ |
| 重複問題文の警告 | 全く同じ問題文が既に存在する場合に警告Toast（保存は可能） |

## 操作フロー

```
管理画面トップ
├→ 「新規追加」→ 入力ダイアログ → 保存 → 一覧に追加
├→ 「編集」→ 既存テキスト入りダイアログ → 保存 → 一覧を更新
└→ 「削除」→ 確認ダイアログ → OK → 一覧から削除（DBから物理削除）
```

## DB操作

```kotlin
// 追加
hiraganaQuestionDao.insert(HiraganaQuestion(text = inputText))

// 編集
hiraganaQuestionDao.update(question.copy(text = newText))

// 削除
hiraganaQuestionDao.delete(question)
```
