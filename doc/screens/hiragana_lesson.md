# ひらがな書き方レッスン画面

## 画面概要
管理画面で登録された短い文（例: 「いぬ」「あおい そら」）を問題として、ひらがなを書き順通りに指でなぞる画面。1文字ずつ順番に入力させる。

## 画面要素

| 要素 | 種類 | 説明 |
|------|------|------|
| 問題文テキスト | TextView | お手本の文字列を上部に表示（例: 「いぬ」） |
| 現在書く文字の強調 | TextView / View | 現在入力対象の文字を大きく・色を変えて表示 |
| 書き順ガイド（薄いお手本） | Custom View | 入力エリア背景に薄くお手本文字を表示 |
| 書き取りキャンバス | Custom View | 指でなぞるCanvas領域 |
| 「もう一度」ボタン | Button | 現在の文字の入力をリセット |
| 「やめる」ボタン | Button | 確認ダイアログを挟んでホーム画面へ戻る |
| 問題番号 | TextView | 例: 「1 / 3」（問題文中の何文字目か） |

## 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 問題文の全文字入力完了 | レッスン結果画面 (`lesson_result`) |
| 「やめる」→確認OK | ホーム画面 (`home`) |

**遷移元:** 教材選択画面 (`lesson_selection`)

## 状態管理

| 状態 | 型 | 説明 |
|------|----|------|
| 現在の問題文 | String | DBからランダム取得した `HiraganaQuestion.text` |
| 現在の文字インデックス | Int | 問題文中の何文字目を入力中か |
| 入力中のストローク | List\<Path\> | 現在の文字に対して描かれたストロークのリスト |
| 入力済み文字の正誤 | List\<Boolean\> | 各文字の判定結果 |

## インタラクション

1. **画面表示**: DBから未受講または正答率の低い問題文を1件取得して表示
2. **指でなぞる**: Canvasにストロークを描画。指を離したタイミングでストロークをリストに追加
3. **書き順判定**: 全ストローク入力後（または一定秒数後）に判定処理を実行
   - 正しい場合: 文字がグリーンにアニメーション → 次の文字へ
   - 誤りの場合: 文字が赤くフラッシュ → 「もう一度」を促す
4. **「もう一度」タップ**: 現在の文字のCanvasと入力ストロークをリセット
5. **全文字完了**: レッスン結果画面へ遷移

## 書き順判定の詳細

書き順判定アルゴリズムの詳細は `features/stroke_order.md` を参照。

## 問題文の取得ロジック

- DBの `HiraganaQuestion` テーブルから1件取得
- 取得優先度: 未受講 > 正答率が低い順
- 問題文は管理画面 (`admin/admin_panel.md`) で登録・管理される
