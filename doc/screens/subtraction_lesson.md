# 引き算レッスン画面

## 画面概要
引き算の問題を1問ずつ出題する画面。10以下の引き算を扱い、正答率に応じて出題する数の大きさが変化する（適応難易度）。答えが負にならない問題のみ出題する。

## 画面要素

| 要素 | 種類 | 説明 |
|------|------|------|
| 問題文 | TextView | 例: 「7 - 3 = ?」大きなフォントで表示 |
| ビジュアル補助 | ImageView × N | 数字に対応する絵を並べ、引く分を×印で表示（任意） |
| 選択肢ボタン × 4 | Button | 正解1つ＋ダミー3つをシャッフルして表示 |
| 問題番号 | TextView | 「1 / 5」形式（1セット5問） |
| 「やめる」ボタン | Button | 確認ダイアログを挟んでホーム画面へ戻る |

## 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 5問終了 | レッスン結果画面 (`lesson_result`) |
| 「やめる」→確認OK | ホーム画面 (`home`) |

**遷移元:** 教材選択画面 (`lesson_selection`)

## 状態管理

| 状態 | 型 | 説明 |
|------|----|------|
| 現在の問題インデックス | Int | 0〜4 |
| 問題リスト | List\<SubtractionQuestion\> | セッション開始時に一括生成 |
| 回答済み結果 | List\<Boolean\> | 各問正解/不正解 |

```kotlin
data class SubtractionQuestion(
    val operandA: Int,  // 2〜上限値（被減数）
    val operandB: Int,  // 1〜operandA（減数）
    val answer: Int,    // operandA - operandB (≥ 0)
    val choices: List<Int>
)
```

## インタラクション

足し算レッスン画面と同一の操作フローを踏む（`addition_lesson.md` 参照）。

## 出題ルール

- 1セット5問一括生成
- 答えが0以上10以下の組み合わせのみ
- 選択肢の範囲: 正解 ±1〜3 の整数4つ（重複・負数を除く）
- 詳細なアルゴリズムは `features/adaptive_difficulty.md` を参照
