# ショップ（ポイント交換）画面

## 画面概要
貯まったポイントを使って、メンダコのご飯・アクセサリー・家具を購入する画面。カテゴリタブで商品を切り替えて表示する。

## 画面要素

| 要素 | 種類 | 説明 |
|------|------|------|
| 現在のポイント表示 | TextView | 画面上部に常時表示「⭐ 123pt」 |
| カテゴリタブ | TabLayout | 「ごはん」「アクセサリー」「かぐ」の3タブ |
| 商品一覧 | RecyclerView | 商品カード（画像・名前・必要ポイント・購入ボタン）のグリッド表示 |
| 購入ボタン | Button | ポイント不足の場合はグレーアウト |
| 「もどる」ボタン | Button / ActionBar | ホーム画面へ戻る |

### 商品カードの要素
- 商品画像（ImageView）
- 商品名（TextView）
- 必要ポイント（TextView）例: 「50pt」
- 購入済みバッジ（TextView/Icon）: 購入済みの場合に「もってる」表示

## 画面遷移

| 操作 | 遷移先 |
|------|--------|
| バック / 「もどる」 | ホーム画面 (`home`) |
| 購入成功ダイアログOK | 同画面（ポイント更新） |

## 状態管理

| 状態 | 型 | 取得元 |
|------|----|--------|
| 現在ポイント | Int | Room DB: `UserState.currentPoints` |
| 商品マスタ一覧 | List\<ShopItem\> | Room DB: `ShopItem` テーブル（マスタデータ） |
| 所持品一覧 | List\<OwnedItem\> | Room DB: `OwnedItem` テーブル |

## インタラクション

1. **画面表示**: ポイント・商品一覧・所持品をDBから取得し表示
2. **タブ切り替え**: 対応するカテゴリの商品のみ表示
3. **購入ボタンタップ**:
   - ポイント不足: ボタンタップ不可（グレーアウト）または「ポイントがたりません」Toast
   - 購入可能: 確認ダイアログ表示「◯◯を50ptで買いますか？」
4. **確認ダイアログOK**:
   - `UserState.currentPoints` を減算
   - `OwnedItem` にレコード追加
   - 商品カードに「もってる」バッジを表示
   - ポイント表示を更新

## 商品カテゴリと仕様

| カテゴリ | 消費方法 | 説明 |
|---------|---------|------|
| ごはん | 一度使うと消費（消耗品） | 「ご飯プレゼント」画面で使用 |
| アクセサリー | 繰り返し使用可 | 「アクセサリー付け替え」画面で装着 |
| かぐ | 繰り返し使用可 | 「家具配置」画面で設置 |
