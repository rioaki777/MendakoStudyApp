# メンダコのお部屋画面

## 画面概要
メンダコとその部屋をインタラクティブに表示するメイン演出画面。メンダコが現在装着しているアクセサリーや部屋の家具が反映された状態で表示される。ここから各カスタマイズ機能へアクセスする。

## 画面要素

| 要素 | 種類 | 説明 |
|------|------|------|
| 部屋背景 | ImageView | 配置済み家具を含む部屋の全体ビュー |
| メンダコ | ImageView | 現在のアクセサリーを装着した状態 |
| 「アクセサリー」ボタン | Button | アクセサリー付け替え画面へ遷移 |
| 「ごはん」ボタン | Button | ご飯プレゼント画面へ遷移（所持ご飯数バッジ付き） |
| 「かぐ」ボタン | Button | 家具配置画面へ遷移 |
| 「もどる」ボタン | Button / ActionBar | ホーム画面へ戻る |

## 画面遷移

| 操作 | 遷移先 |
|------|--------|
| 「アクセサリー」タップ | アクセサリー付け替え画面 (`accessories`) |
| 「ごはん」タップ | ご飯プレゼント画面 (`food`) |
| 「かぐ」タップ | 家具配置画面 (`furniture`) |
| バック / 「もどる」 | ホーム画面 (`home`) |

## 状態管理

| 状態 | 型 | 取得元 |
|------|----|--------|
| 装着中アクセサリー | List\<AccessoryId\> | Room DB: `UserState.equippedAccessories` |
| 配置済み家具 | List\<FurniturePlacement\> | Room DB: `FurniturePlacement` テーブル |
| 所持ご飯数 | Int | Room DB: `OwnedItem` (category=FOOD) の個数 |

## インタラクション

1. **画面表示**: DBからメンダコの状態・家具配置を取得してレンダリング
2. **各ボタンタップ**: 対応するカスタマイズ画面へ遷移
3. **各サブ画面から戻ったとき**: `onResume` でDBを再取得し、変更を反映して再描画
4. **「ごはん」ボタン**: 所持ご飯数が0の場合はバッジ「0」表示、タップ時「ごはんがありません」Toast

## 描画方針

- メンダコ画像はベース画像に対してアクセサリー画像をオーバーレイするレイヤー合成
- 部屋背景は固定の背景画像に家具画像を各配置座標に描画する（FrameLayoutまたはCustom View）
