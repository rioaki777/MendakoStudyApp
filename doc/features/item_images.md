# アイテム画像 要求仕様

ショップで販売・使用するアイテム（アクセサリー・ごはん・かぐ）の画像仕様。
メンダコのへや・ショップ画面に表示するために使用する。

---

## 共通仕様

| 項目 | 内容 |
|---|---|
| フォーマット | 透過PNG（.png） |
| 解像度 | xxhdpi：192×192px（必須）、xxxhdpi：256×256px（任意） |
| 配置先 | `res/drawable-xxhdpi/`（xxxhdpiは `res/drawable-xxxhdpi/`） |
| キャンバス | 192×192px（各アイテム共通） |
| 背景 | 透明（アイテムの輪郭以外は透過） |
| タッチ感 | 子ども向けのやわらかく丸みのあるデザイン |
| 色味 | 明るくポップ。アプリのメインカラー（青・オレンジ・クリーム）と馴染む配色 |

> キャンバスサイズは全アイテムで統一する。
> ショップカード（56dp相当）とアクセサリーoverlay（24〜60dp相当）の両方に使われるため、小サイズでも視認できるデザインにすること。

---

## アクセサリー（ACCESSORY）

メンダコの体に重ねて表示するオーバーレイ画像。
FrameLayout 内で body・eyes・mouth の上に重ねて描画される。

### 表示位置

| ファイル名 | アイテム名 | 表示位置 | サイズ（dp） |
|---|---|---|---|
| `accessory_hat.png` | ぼうし | 頭の上（top \| center_horizontal） | 40×24dp |
| `accessory_scarf.png` | マフラー | 首元（center） | 60×16dp |
| `accessory_ribbon.png` | リボン | 頭の右上（top \| end） | 24×24dp |

### デザイン要件

#### `accessory_hat.png`（ぼうし）
- メンダコの頭頂部に乗る帽子
- キャンバス下端付近に帽子の形を描く（重ねたとき頭上に見えるように）
- 色味の参考：濃い紫系（`#6A0DAD` 近辺）
- 帽子の種類は丸いベレー帽・三角帽子・ニット帽など、子どもが好む形であればOK
- キャンバス全体に対してパーツは中央下寄りに配置し、上部は透明

#### `accessory_scarf.png`（マフラー）
- メンダコの首に巻かれているマフラー
- 横長の帯状、もしくはふんわり巻かれた形
- キャンバス中央に配置（上下は透明）
- 色味の参考：オレンジ系（`#FF6B35` 近辺）

#### `accessory_ribbon.png`（リボン）
- メンダコの頭右上に付くリボン
- かわいらしい蝶々結びのリボン
- キャンバス左下寄りに配置（重ねたとき頭右上に見えるように）
- 色味の参考：ピンク系（`#FF69B4` 近辺）

### 実装上の注意

アクセサリー画像はメンダコ本体（120dp×120dp FrameLayout）に重ねて表示する。
各画像の表示サイズは dp で固定されているため、**パーツはキャンバスの一部だけに描き、余白は透明にする**こと。
重ねたときに自然に見えるよう、メンダコ本体の位置を意識してパーツの配置場所を決める。

```
FrameLayout（120×120dp）のイメージ:
┌─────────────────────────┐
│     [hat ここに帽子]     │ ← accessory_hat (40×24dp, top|center)
│                         │
│   ┌───────────────┐    │
│   │  mendako_body │    │
│   │  (青いメンダコ│    │
│   │  の体)        │    │
│   └───────────────┘    │
│  [scarf ─────── scarf]  │ ← accessory_scarf (60×16dp, center)
│                      [♥]│ ← accessory_ribbon (24×24dp, top|end)
└─────────────────────────┘
```

## ごはん（FOOD）

ショップカード・ごはん画面の一覧に表示するアイテム画像。
メンダコに「たべさせる」アイテムとして使われる。

### ファイル一覧

| ファイル名 | アイテム名 | 色味の参考 |
|---|---|---|
| `food_fish.png` | おさかな | 青・銀系（`#5B9BD5` 近辺） |
| `food_crab.png` | かに | 赤・オレンジ系（`#E53935` 近辺） |
| `food_shrimp.png` | えび | ピンク・サーモン系（`#EF9A9A` 近辺） |

> 現在の実装では3アイテムが同じプレースホルダー画像（`item_food.xml`）を共有している。
> 個別画像を追加する際は、各アイテムの `imageResName` カラムの値をファイル名に合わせて更新すること（SeedData.kt参照）。

### デザイン要件

- 正面または斜め上から見た食べ物のイラスト
- シンプルで子どもが一目でわかる形
- 丸みのある輪郭、なめらかな線
- サイズの目安：キャンバス（192px）の70〜80%を食べ物が占める、周囲は透明

#### `food_fish.png`（おさかな）
- 魚全体のシルエット（横向き）
- ひれ・うろこなど特徴的なパーツを含む
- 色：青・水色・銀色など魚らしい色

#### `food_crab.png`（かに）
- カニを正面から見たシルエット
- 大きなハサミを左右に広げた形
- 色：赤・オレンジ系

#### `food_shrimp.png`（えび）
- エビを横向きに描いたシルエット
- 丸まった体・長い触角など特徴的な形
- 色：ピンク・サーモン系（茹でたえびのイメージ）

## かぐ（FURNITURE）

かぐ画面の部屋エリアに配置・表示されるアイテム画像。
ドラッグ＆ドロップで自由に動かせるため、**どの角度から見ても形がわかる**デザインが望ましい。

### ファイル一覧

| ファイル名 | アイテム名 | 色味の参考 |
|---|---|---|
| `furniture_table.png` | テーブル | 茶・木目系（`#8B6347` 近辺） |
| `furniture_bed.png` | ベッド | 白・ベージュ系（枠は茶） |
| `furniture_sofa.png` | ソファ | 青・グレー系（`#5B9BD5` 近辺） |

> 現在の実装では3アイテムが同じプレースホルダー画像（`item_furniture.xml`）を共有している。

### デザイン要件

- 斜め上（俯瞰・等角投影風）から見た家具のイラスト
- 部屋に置かれているように見える視点が望ましい
- 輪郭はくっきり、影は薄めのドロップシャドウ程度
- 表示サイズが 56dp（約168px @ xxhdpi）程度のため、細かいテクスチャは不要

#### `furniture_table.png`（テーブル）
- 小さな丸テーブルまたは四角いローテーブル
- 足が見える斜め俯瞰視点
- 色：木目の茶色系

#### `furniture_bed.png`（ベッド）
- シングルベッド（枕・布団付き）
- 斜め上から見た視点
- 枠は茶・木目、布団は白・薄い青など

#### `furniture_sofa.png`（ソファ）
- 2〜3人掛けソファまたはひとりがけソファ
- 斜め前から見た視点（正面気味でも可）
- 色：青・グレー・ベージュなど

## ファイル配置まとめ

```
res/
  drawable-xxhdpi/          ← 192×192px（必須）
    accessory_hat.png
    accessory_scarf.png
    accessory_ribbon.png
    food_fish.png
    food_crab.png
    food_shrimp.png
    furniture_table.png
    furniture_bed.png
    furniture_sofa.png
  drawable-xxxhdpi/          ← 256×256px（任意）
    （同名ファイル）
```

## SeedDataの更新（画像追加後に必要）

個別の画像ファイルを追加したら `SeedData.kt` の `imageResName` を更新する。

```kotlin
// 変更前（プレースホルダー）
ShopItem(1, "おさかな", "FOOD", 20, "item_food"),
ShopItem(2, "かに",    "FOOD", 30, "item_food"),
ShopItem(3, "えび",    "FOOD", 40, "item_food"),

// 変更後（個別画像）
ShopItem(1, "おさかな", "FOOD", 20, "food_fish"),
ShopItem(2, "かに",    "FOOD", 30, "food_crab"),
ShopItem(3, "えび",    "FOOD", 40, "food_shrimp"),
```

アクセサリー・かぐも同様に `imageResName` を対応するファイル名に変更する。

DBはすでにシード済みの場合、バージョン番号を上げてマイグレーションするか、アプリをアンインストールして再インストールすること。
